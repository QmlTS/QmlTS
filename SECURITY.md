# Security Policy

## Supported Versions

We currently only provide security updates and patches for the latest code on the `main` branch of the project. Older released versions are not supported for security updates.

| Version / Branch      | Supported          |
| --------------------- | ------------------ |
| Latest (main branch)  | :white_check_mark: |
| Older versions/tags   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in this project, please report it to us privately using GitHub's built-in Private Vulnerability Reporting tool.

1. Go to the **Security** tab of this repository.
2. Click on **Advisories** in the left sidebar.
3. Click the **Report a vulnerability** button.
4. Provide a descriptive summary and details of the vulnerability, including steps to reproduce.

We take all security reports seriously and will review and respond to reports as promptly as possible. Once an issue is resolved, we may publish a security advisory and, with your permission, credit you for the discovery using your preferred attribution details.

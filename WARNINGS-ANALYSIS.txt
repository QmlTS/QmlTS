# Analysis of 215 Remaining Warnings in QmlTS Full-Registry DSL Generation

## Executive Summary

The 215 warnings are all generated by the **TypeMapper.fallback()** method (type-mapper.ts:213-220) when a QML type cannot be resolved through any of 8 resolution strategies. These are not errors—the generator falls back to QmlValue (a catch-all type union in runtime/types.ts:86-106) which is semantically correct.

---

## Exact Warning Structure

### Generation Point
**File**: src/dsl/generator/type-mapper.ts, lines 213-220
**Method**: TypeMapper.fallback(qmlType: string)

The fallback method pushes a diagnostic with:
- **level**: 'warning'
- **message**: Unknown QML type "TYPE_NAME", falling back to QmlValue
- **result**: Returns QmlValue (catch-all type) and adds it to runtimeImports

### Aggregation Point
**File**: src/dsl/generator/generator.ts, line 91
- Diagnostics collected from mapper.getDiagnostics() after type mapping
- Returned in GeneratorResult

---

## Type Resolution Pipeline (mapType method, lines 40-97)

The TypeMapper attempts 8 strategies in order:

1. **Static QML_TYPE_MAP lookup** (lines 42-46)
   - Primitives: int, double, bool, string -> number, boolean, string
   - C++ typedefs: qint8, quint64, etc -> number
   - Semantic aliases: color -> QmlColor, url -> QmlUrl
   - Structured values: point -> QmlPoint, vector3d -> QmlVector3d
   - Value unions: var, variant -> QmlValue

2. **List type matching** (lines 49-54)
   - Regex: /^(?:QList|list)<(.+)>$/
   - Examples: list<string> -> string[], QList<Entity> -> EntityBuilder[]
   - Issue: Bare 'list' (without angle brackets) NOT matched

3. **QQmlComponent hardcoded special case** (lines 57-60)
   - QQmlComponent -> QmlComponent

4. **Analyzed Registry lookup** (lines 63-66)
   - Looks up by C++ qualified name
   - Returns EntityBuilder, SingletonInstance, etc.
   - Issue: Only works if type is in analyzed registry

5. **C++ namespace stripping** (lines 69-74)
   - Strips :: prefix and retries
   - Qt::Alignment -> Alignment, then retries

6. **Pointer suffix stripping** (lines 77-81)
   - Removes * and retries
   - QEntity* -> QEntity

7. **Grouped surface lookup** (lines 84-87)
   - Checks analyzed.groupedSurfaces map

8. **Enum index lookup** (lines 90-93)
   - Returns QmlEnumToken if found in enum index

9. **Fallback** (line 96)
   - Generates warning and returns QmlValue

## Categories of Unresolved Types (215 Total)

### Category 1: Unqualified Qt3D Types (77%)
**Count**: ~165 warnings
**Root Cause**: Properties reference short names (QEntity) instead of qualified names (Qt3DCore::QEntity)

Examples:
- QEntity: 100 warnings (8 properties, 1 return, 6 method params, 8 signal params)
- QAttribute: 8 warnings (33 properties, 3 method params, 1 signal param)
- QGeometry: 7 warnings (6 properties, 7 method params, 6 signal params)
- QBuffer: 1 warning

**Used in modules**: Qt3D.Animation, Qt3D.Core, Qt3D.Extras, Qt3D.Logic, Qt3D.Render, QtQuick.Scene2D, QtQuick.Scene3D

**Why existing strategy fails**: Types ARE in registry as Qt3DCore::QEntity, but referenced as unqualified QEntity. Namespace stripping helps with Qt::Alignment -> Alignment, but doesn't reverse-lookup by short name.

### Category 2: Bare 'list' type (3.7%)
**Count**: 8 warnings
**Root Cause**: Registry contains bare 'list' without <...> syntax

Examples:
- LevelOfDetailLoader.sources: list (C++ QVariantList)
- BarSet.values: list (C++ QVariantList)
- BoxSet.values: list (C++ QVariantList)
- geoPath.path: list (C++ QVariantList)

**Why not matched**: Regex requires angle brackets. Bare list is unqualified variant.

**Solution**: Add check for bare list before regex:
`
if (qmlType === 'list') return 'QmlValue[]';
`

### Category 3: Data Container Types (11.6%)
**Count**: ~25 warnings
**Root Cause**: Internal C++ structs for data interchange, not creatable QML types

Examples:
- QScatterDataArray, QBarDataArray, QSurfaceDataArray (QtGraphs)
- QLinearGradient: 4 warnings (9 properties, 3 method params, 12 signal params)
- QMatrix3x3: 4 warnings
- QTimeZone: 2 warnings

**Why not resolved**: Not exported as QML types (exports.length = 0), classified as non-creatable or internal

### Category 4: Graphics/Rendering Types (8%)
**Count**: ~17 warnings
**Root Cause**: Base classes and infrastructure for render pipeline

Examples:
- QEffect: 2 properties, 2 method params, 2 signal params
- QAbstractTexture: 17 properties, 16 method params, 16 signal params (heavily used!)
- QRenderTarget: 8 properties, 3 method params, 3 signal params
- QRenderTargetOutput: 5 properties, 2 method params, 2 signal params
- QFrameGraphNode, QShaderProgram, QAbstractClipBlendNode

### Category 5: Input Device Types (1.9%)
**Count**: 4 warnings
**Root Cause**: Input handler base types

Examples:
- QAbstractPhysicalDevice
- QKeyboardHandler, QKeyboardDevice, QMouseDevice

### Category 6: Domain-Specific Types (3.7%)
**Count**: ~8 warnings

Examples:
- QPlace, QPlaceCategory (QtLocation)

- QScxmlDataModel (QtScxml)

### Category 7: Style System Types (0.9%)

## Mapping Improvement Strategy

### Quick Win (Adds ~8-10 resolved types, removes ~8 warnings)
**Bare 'list' handling**
`
if (qmlType === 'list' || qmlType === 'QVariantList') {
  return 'QmlValue[]';
}
`

### High-Impact (Reduces ~120 warnings, 56%)
**Qualified name reverse-lookup for Qt3D and other modules**

When short name lookup fails but no :: in name:
`
Build short-name index: QEntity -> Qt3DCore::QEntity
Use in mapType() after namespace stripping fails
`

This alone would eliminate ~100 QEntity warnings and ~8 QAttribute warnings.

### Medium-Impact (Reduces ~30-40 warnings, 14-19%)
**Categorical mapping** for heavily-used non-creatable types:
- QLinearGradient: 9 properties + 15 params -> map to custom QmlGradient or QmlValue
- QAbstractTexture: 17 properties + 32 params -> map to custom QmlTexture or QmlValue
- QMatrix3x3: Custom QmlMatrix3x3 or QmlValue
- QTimeZone: QmlValue
- QPlace, QPlaceCategory: QmlValue
- QVideoFrame: QmlValue

### Remaining (~40-50 warnings)
**Long-tail domain-specific and rarely-used types**
- Leave as QmlValue (semantically correct catch-all)
- Or create domain-specific mapping if needed

---

## Key Insights

1. **Registry has the types** - They're not missing, they're just under qualified names (Qt3DCore::QEntity instead of QEntity)

2. **Property usage dominates** - 79% of warnings come from property types, not method signatures

3. **Qt3D is the main issue** - 56% of warnings (120) are Qt3D unqualified types that exist in Qt3D.Core

4. **Bare 'list' is easy fix** - 8 warnings from a simple type map addition

5. **Graphics types are high-value targets** - QAbstractTexture alone affects 33 locations (17 properties + 16 params)

6. **Fallback is sound semantics** - QmlValue union accepts all valid QML values, so no functionality is lost
